Admin Portal Overview

The Django administration interface provides powerful tools for configuring and monitoring your chatbot. Each section serves a specific purpose in creating an effective chatbot experience.

Chat Infos

This section contains general information about chat sessions, such as user identifiers, session durations, and basic metrics.

Strategic Use:

  • Monitor chat volume by time of day/week to identify peak usage periods
  • Track average session duration to understand engagement levels
  • Identify drop-off points where users abandon conversations
  • Look for patterns in user behavior that might indicate different user groups
  • Set up regular reviews of key metrics like session completion rates

Implementation Tasks:

  1. Establish baseline metrics for current chatbot performance
  2. Create a weekly review process for chat analytics
  3. Develop a tracking system for identifying trends over time

Chatbot Automated Answers

This section allows you to create pre-defined responses for common questions, ensuring consistency and appropriate reading level.

Strategic Use:

  • Create simplified, encouraging answers for frequently asked questions
  • Ensure all responses maintain 5th-grade reading level
  • Include clear action steps in every response
  • Add encouragement in every response

Implementation Tasks:

  1. Identify the 20 most common questions from conversation logs
  2. Create standardized responses using the template provided in the Content Format Templates section
  3. Test responses with sample users before implementation
  4. Gradually expand to cover more specific scenarios

Example Format:

Question trigger: "How do I make a resume?"

Response: "Making a resume is easy! Click the 'Resume Builder' button at the top of your screen. It will guide you step by step. You can save your work and come back to it later. Would you like to start your resume now? I'm here to help if you get stuck."

Chatbot Automated Questions

This section allows you to program proactive questions that the chatbot can ask users, particularly valuable for users who need extra encouragement.

Strategic Use:

  • Create engagement questions triggered by time spent on a page without action
  • Develop feature discovery questions to introduce unused features
  • Design progress celebration questions to acknowledge completed steps
  • Implement re-engagement questions for returning users

Implementation Tasks:

  1. Identify key moments in the user journey where users commonly get stuck
  2. Create proactive questions for these moments using the templates provided
  3. Test questions with sample users to ensure they're helpful, not intrusive
  4. Monitor impact on user engagement and task completion

Example Types:

  • Engagement: "I see you're looking at job listings. Would you like help filtering these jobs to find the best match?"
  • Feature Discovery: "Did you know you can take a quick survey to find jobs that match your interests?"
  • Progress Celebration: "Great job finishing your profile! Would you like to look at some jobs that match your skills?"
  • Re-engagement: "Welcome back! Last time you started your resume but didn't finish. Would you like to continue where you left off?"

Chatbot Feedback Ratings

This section stores user feedback about the chatbot's performance, providing valuable insights for improvement.

Strategic Use:

  • Create categories for feedback (e.g., clarity, helpfulness, accuracy)
  • Look for patterns in negative feedback
  • Identify which topics receive the most positive/negative ratings
  • Prioritize improving responses with consistently low ratings
  • Analyze high-rated responses to identify effective communication patterns

Implementation Tasks:

  1. Establish a feedback categorization system
  2. Create a monthly review process to address feedback trends
  3. Develop a prioritization framework for addressing negative feedback
  4. Consider implementing a mechanism to ask for more details after negative ratings
