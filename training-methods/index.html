Training Methods

The ChatArbor Training interface offers three complementary methods to train your chatbot, each with specific advantages and use cases.

Train with Files

This method allows you to upload structured data files to train your chatbot.

Strategic Use:

  • Upload comprehensive datasets covering core topics
  • Maintain consistent structure across training files
  • Update quarterly with new information and refined responses

File Formats and Best Practices:

  • CSV files: Excellent for structured Q&A pairs
  • JSON files: Maintain more complex hierarchical relationships
  • TXT files: Work well for narrative content and examples

Content Organization:

  • Question/Answer pairs (most common format)
  • Topic/Explanation pairs
  • User scenario/Response examples

Medicaid Member-Specific Content:

  • Simplified explanations of employment terms
  • Step-by-step guides for using portal features
  • Common barriers and solutions
  • Success stories written at appropriate reading levels

RAG Implementation:

This training method uses a technique called RAG (Retrieval-Augmented Generation).

RAG is like giving your chatbot a personalized reference library. When a user asks a question, the system first searches this library for relevant information (Retrieval), then uses that information to create a response (Generation). It's similar to how you might look up facts in a book before answering a complex question, rather than relying solely on memory.

RAG is particularly valuable for your use case because it allows the chatbot to provide accurate, specific information about your services and portal features without requiring that information to be embedded in the base AI model.

Implementation Tasks:

  1. Create a file structure for organizing training content
  2. Develop templates for different types of training files
  3. Begin with core topics (resume building, job search, etc.)
  4. Establish a quarterly review and update process

Train with Website URLs

This method allows the system to crawl and extract information from websites you specify.

Strategic Use:

  • Add external resources that complement your core content
  • Update monthly as new resources become available

URL Selection Strategy:

  • Start with your own portal's help documentation
  • Include career resources written at appropriate reading levels
  • Add government resources about Medicaid benefits and employment
  • Consider disability employment resources

Content Verification:

  • Before adding URLs, verify the content is accurate and at an appropriate reading level
  • Check that information aligns with your services and doesn't create confusion

Incremental Addition:

  • Add URLs in batches and test the chatbot's responses after each addition
  • This helps identify which sources improve performance and which might introduce confusion

Implementation Tasks:

  1. Create a spreadsheet to track potential URLs for training
  2. Evaluate each URL for reading level and relevance
  3. Develop a testing protocol for after adding new URLs
  4. Establish a monthly URL verification process

Train with Text

This direct input method is ideal for quick additions and testing.

Strategic Use:

  • Use for immediate adjustments based on user feedback
  • Test new approaches before adding to permanent files

Use Cases:

  • Rapidly respond to emerging questions or issues
  • Test different phrasings to find what works best
  • Add time-sensitive information (like program deadlines)
  • Create examples of ideal tone and reading level

Content Structure:

  • Format as clear question/answer pairs
  • Include variations of how users might ask the same question
  • Provide examples of both the question and ideal response

Implementation Tasks:

  1. Create a process for quickly addressing emerging issues
  2. Develop a testing protocol for new response approaches
  3. Establish guidelines for when to move text training into permanent files

Integrating Training Methods

For optimal results, use all three methods in coordination:

Foundation Building (Files)

  • Create comprehensive files covering all core topics
  • Update quarterly with new information and refined responses

Knowledge Expansion (URLs)

  • Add external resources that complement your core content
  • Update monthly as new resources become available

Rapid Response (Text)

  • Use for immediate adjustments based on user feedback
  • Test new approaches before adding to permanent files
